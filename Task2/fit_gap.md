# Fit/Gap анализ перехода компании "Diamond Universe" для перехода на online формат продаж

> Анализ производится для двух решений:
> 	1. Промежуточное решение (минимум доработок и интеграций с ИС);
> 	2. Целевое решение (полная автоматизация );

## Список систем 

### ИС "Алмазы" (существующая)

| **Функционал**    | **Тип функционала**    | **Необходимые доработки**|   
|---------------|--------|----------------------|
| Учет алмазов  | Существующий   | Доработка не требуется  |  
| Подготовка учетных ведомостей   | Существующий   | Для промежуточного решения выгрузка ведомостей в Excel фаил заранее установленного формата  |  
| Отбор алмазов для аукциона  | Новый  |  Для промежуточного решения ведется реестр в Excel документе. Для целевого решения необходимо шаг,фильтрацию и новый бизнесс процесс "Отбор для аукциона"  |

### ИС "Аукцион" (существующая)

| **Функционал**    | **Тип функционала**    | **Необходимые доработки** |

| Проведение offline торгов | Существующий | Доработки не требуются |

| Передача резуьтатов торгов в смежные ИС : "Алмазы", "Логистика", "Бухгалетрия" | Новый | Промежуточное решение: передача результатов в Excel файле. Целевое решение: автоматическа передача данных |

## Список интеграций

### 2.1. ИС «Алмазы» ->  ИС «Логистика»

- **Тип:** существующая интеграция, требует расширения для нового маршрута.
- **Реализация:** автоматическая (как сейчас); для нового маршрута «в лабораторию» — **промежуточное:** ручная передача заявки по Excel-шаблону, **целевое:** автоматически.
- **Данные:** существующие (паспорт партии, маршрут, ответственные); **новые** — тип маршрута «оцифровка», адрес лаборатории, плановые сроки возврата.

### 2.2. ИС «Лаборатория» -> ИС «Аукцион»

- **Тип:** **новая интеграция**.
- **Реализация:** **промежуточное** — вручную (выгрузка из ИС «Лаборатория» → Excel/файлы по шаблону → загрузка/публикация в облако и далее в ИС «Аукцион»); **целевое** — автоматически по API через выделенный защищённый канал.
- **Данные:** **новые** — цифровые описания алмазов (фото, видео, 3D-модели, отчёты по размеру, чистоте, флуоресценции, внутренним напряжениям, трещинам, включениям), привязка к идентификаторам из ИС «Алмазы».

### 2.3. ИС «Лаборатория» -> ИС «Бухгалтерия»

- **Тип:** **новая интеграция**.
- **Реализация:** **промежуточное** — вручную (акты и счета приходят от подрядчика по почте, бухгалтер регистрирует в системе); **целевое** — автоматически по API/EDI.
- **Данные:** **новые** — акты выполненных работ по оцифровке партий, счета, реестр обработанных алмазов с идентификаторами.

### 2.4. ИС «Логистика» -> ИС «Бухгалтерия»

- **Тип:** существующая интеграция.
- **Реализация:** автоматическая. Доработок не требуется, тарифицируется новый тип маршрута «оцифровка».
- **Данные:** существующие (акты и счета перевозчиков), объём данных вырастет за счёт новых рейсов «офис ↔ лаборатория».

### 2.5. ИС «Алмазы» -> ИС «Лаборатория» *(не указана в исходных данных, но необходима — это GAP)*

- **Тип:** **новая интеграция**.
- **Реализация:** **промежуточное** — вручную (Excel-реестр идентификаторов передаётся с партией); **целевое** — автоматически по API.
- **Данные:** **новые** — реестр идентификаторов алмазов в партии, требования к оцифровке.

### 2.6. ИС «Алмазы» -> ИС «Аукцион» *(GAP для онлайн-формата)*

- **Тип:** существующая интеграция, требует доработки под онлайн-сценарий.
- **Реализация:** **промежуточное** — вручную через Excel-выгрузку отобранных лотов; **целевое** — автоматически с признаком «онлайн-аукцион».
- **Данные:** существующие (лоты, идентификаторы, оценочная стоимость); **новые** — флаг канала продажи (офлайн/онлайн).

### 2.7. ИС «Аукцион» -> ИС «Алмазы» / ИС «Логистика» / ИС «Бухгалтерия» *(GAP для онлайн-формата)*

- **Тип:** существующие интеграции для офлайн-аукциона; для онлайн-формата требуют доработки.
- **Реализация:** **промежуточное** — результаты онлайн-торгов передаются вручную (Excel-протокол по победителям); **целевое** — автоматически.
- **Данные:** существующие (протокол победителей, цены, реквизиты покупателей, адреса доставки).

### 2.8. ИС «Аукцион» <--> Облачные сервисы провайдера *(только для промежуточного решения)*

- **Тип:** **новая интеграция** (временная).
- **Реализация:** вручную/полуавтоматически (загрузка файлов с описаниями алмазов в облачное хранилище, рассылка ссылок клиентам).
- **Данные:** **новые** — пакеты цифровых описаний для публикации; в целевом решении заменяются на размещение в собственном контуре компании.
